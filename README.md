# Workflow

A Claude Code **skill marketplace**. It publishes three plugins that turn a fuzzy idea into shipped, spec-checked code — and the skills that author and maintain those skills. The product is not an app; it is the skills and *how they hand off to each other*.

## Quickstart

1. Add the marketplace, then install the kit you want — see [Installation](#installation).
2. Building a feature? Run the gated chain: `grilling` → `writing-specs` → `writing-plans` → implement test-first → `spec-drift-audit`.
3. Just learning a topic, or polishing prose? Reach for [learning-kit](plugins/learning-kit/README.md) or [craft-kit](plugins/craft-kit/README.md).

## Installation

### Claude Code

```text
/plugin marketplace add enkeapple/obilyk-skills
/plugin install sdd-kit@obilyk-skills
```

Swap `sdd-kit` for `learning-kit` or `craft-kit` to install another plugin. Each plugin is self-contained — install only the ones you need.

## Philosophy

- **Predictability over cleverness.** A skill exists to make the agent take the same *process* every run.
- **Test-first, always.** No skill ships without a failing subagent pressure run first (RED), then compliance with the skill (GREEN).
- **Agnostic by default.** A skill never hard-codes one project's stack, paths, or commands — the consumer repo fills the specifics.
- **Derived, not hand-curated.** Catalogs and indexes are generated from frontmatter on disk, so they can be audited for drift instead of trusted.

## Contributing

Author or change a skill through `writing-skills` (RED → GREEN → REFACTOR → VALIDATE). A change is done only when the validators pass (frontmatter ≤1024, name regex, links resolve, fences balanced, word count) **and** an independent subagent run confirms the behavior. Regenerate this index and the per-plugin catalogs with `bootstrapping-readme`; check them for drift with `auditing-readme`.
