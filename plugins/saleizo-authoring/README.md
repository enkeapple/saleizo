# Authoring Kit

Test-first authoring of the framework's own artifacts. Every skill here drives a RED ‚Üí GREEN ‚Üí REFACTOR ‚Üí VALIDATE loop: you create a skill, hook, rule, lesson, or ADR only after a baseline check fails, then prove it holds.

## Quickstart

- Writing or changing a skill? Reach for `writing-skills`. A Claude Code hook? `writing-hooks`. A project rule? `writing-rules`.
- Capture a recurring failure as a lesson with `writing-lessons`; record an architectural decision with `writing-adrs`.
- Install it first ‚Äî see [Installation](#installation).

## How it works

Each skill is invoked by naming the authoring task ("write a skill", "add a hook", "record this decision"). The skill supplies the methodology ‚Äî the failing-test-first discipline, the validators, and the cold-reviewer gate ‚Äî so the artifact is verified, not just written.

## What's Inside

- **writing-skills** ‚Äî author, edit, or validate a skill in this repo test-first (RED ‚Üí GREEN ‚Üí REFACTOR ‚Üí VALIDATE).

## Philosophy

This repo exists to practice test-first authoring on itself. These skills are how that discipline is applied to skills, hooks, rules, lessons, and decisions ‚Äî no artifact ships because the markdown exists; it ships because a baseline failed without it and a validated run confirms it.
