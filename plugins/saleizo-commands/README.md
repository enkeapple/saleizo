# Commands Kit

Short, user-typed alias commands. Each is a thin `disable-model-invocation` facade that delegates to exactly one canonical SDD skill and forwards `$ARGUMENTS` — it holds no logic of its own, so the behavior lives in the target skill, not here.

## Quickstart

- Type `/sdd <idea or ticket>` to run the whole gated pipeline, or reach for a single phase: `/grill`, `/spec`, `/audit`, `/adr`.
- Install it first — see [Installation](#installation). The canonical skills ship in `saleizo-core` / `saleizo-authoring`; install those too.

## How it works

Each alias is invoked by typing its slash form (`/sdd`, `/grill`, …). The body names its one canonical skill and passes your input straight through — the alias is a stable entry point, not a second implementation. Because the target is resolved by bare skill name, the canonical skill can live in any installed plugin.

## Installation

### Claude Code

```text
/plugin marketplace add enkeapple/saleizo
/plugin install saleizo-commands@saleizo
```

## What's Inside

- **sdd** → `sdd-lifecycle` — run the full gated SDD pipeline end-to-end.
- **grill** → `grilling` — turn a fuzzy idea into a shared, concrete design.
- **spec** → `writing-specs` — turn an approved design into a concrete spec.
- **audit** → `verifying-implementation` — check shipped code against an approved spec.
- **adr** → `writing-adrs` — record an architectural decision as an immutable ADR.

## Philosophy

Aliases keep the muscle-memory entry points (`/sdd`, `/grill`) stable while the real skills evolve and move between plugins. A command never duplicates logic — it forwards to the canonical skill so there is exactly one place the behavior is defined.

## Contributing

These are facades — change the behavior in the canonical skill via `writing-skills`, not here. Regenerate this catalog with `bootstrapping-readme`; check it for drift with `auditing-readme`.
