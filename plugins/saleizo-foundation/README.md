# Foundation Kit

Set up the SDD framework in a consumer repo and keep its foundations true. Each concern comes as a **bootstrap** skill (create the doc from scratch) paired with an **audit** skill (check the existing doc for drift), plus the one-time adoption flow and a telemetry review.

## Quickstart

- Onboarding a fresh repo? Run `adopting-framework` (copy → routing → bootstraps → verify).
- Create the foundations: `bootstrapping-claude-md`, `bootstrapping-glossary`, `bootstrapping-readme`.
- Keep them true: `auditing-claude-md`, `auditing-glossary`, `auditing-readme`, `auditing-hooks`, `auditing-conflicts`.
- Install it first — see [Installation](#installation).

## How it works

Each skill is invoked by naming the foundation task ("set up CLAUDE.md", "audit the rules", "is the README accurate"). Bootstrap skills generate a doc from disk facts; audit skills verify an existing doc against current code and report drift before you fix it.

## Installation

### Claude Code

```text
/plugin marketplace add enkeapple/saleizo
/plugin install saleizo-foundation@saleizo
```

## What's Inside

- **adopting-framework** — the ordered post-install bootstrap of a fresh consumer repo (copy → symlinks → bootstraps → routing → verify).
- **bootstrapping-claude-md** / **auditing-claude-md** — create / drift-check the two CLAUDE.md files (entry point + operating manual).
- **bootstrapping-glossary** / **auditing-glossary** — create / drift-check the base domain rules (glossary + framework charter).
- **bootstrapping-readme** / **auditing-readme** — generate / drift-check the README skills catalog from frontmatter on disk.
- **auditing-hooks** — find drift between hook scripts and their wiring (orphan / dangling / broken-symlink / event-matcher / fixture-gap).
- **auditing-conflicts** — find conflicts and contradictions across skills, rules, and routing.
- **reviewing-telemetry** — review the framework's own routing/telemetry health and produce a fixed-shape triage digest.

## Philosophy

The framework only works if its foundations stay true to the code. Pairing every bootstrap with an audit makes drift a first-class, detectable state instead of a silent rot — the repo's CLAUDE.md, glossary, README, hooks, and routing are checkable artifacts, not write-once documents.
