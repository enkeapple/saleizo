# Docs Kit

Skills for maintaining a consumer repo's project documentation as manifest-declared views over machine-readable sources: set the docs up from scratch, audit them for drift and fix them, and extend the system with new doc types. A per-project `.claude/docs-manifest.json` maps every doc; a contract declares how each managed block is derived; human prose outside the managed blocks is never touched.

## Quickstart

- No docs yet? `scaffolding-docs` — detect sources, propose a manifest at an approval gate, generate the managed docs.
- Docs might be stale? `auditing-docs` — re-derive machine blocks, flag stale prose, delegate external docs, fix the drift.
- Need a doc type the built-ins don't cover? `extending-docs` — author a new derivation contract test-first.
- Install it first — see [Installation](#installation).

## How it works

Each doc is one manifest entry: `managed` (a contract renders a block from a `source`), `external` (owned by another plugin's skill, audit delegates to it), or `prose` (freeform). A contract's `auditTier` — machine, prose, or hybrid — decides how it is checked: machine blocks are re-derived and diffed; prose/hybrid tiers get a content-hash staleness signal. Eight universal contracts ship built in (`commands`, `environment`, `dependencies`, `testing`, `architecture`, `conventions`, `data-model`, `api-surface`); `extending-docs` adds more.

## What's Inside

- **scaffolding-docs** — set up documentation from scratch: detect candidate sources, gate ambiguous bindings, propose a manifest, generate each managed doc with its rendered block and `rev`.
- **auditing-docs** — audit docs for drift and apply fixes: re-derive machine blocks, flag stale prose, delegate external docs (fail loud on an uninstalled owner), report a recommended disposition per finding through one batched picker.
- **extending-docs** — add a new doc type by authoring a derivation contract test-first (schema-valid header + scaffold→audit round-trip).

## Philosophy

Documentation rots because nothing re-derives it from the truth. This kit makes the derivation explicit — a source, a canonical render, a content hash — so an audit can tell a stale doc from a current one, and regeneration touches only the machine-owned regions while leaving human prose alone.
