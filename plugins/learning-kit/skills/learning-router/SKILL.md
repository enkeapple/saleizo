# Learning Router

Nine learning skills exist, all user-invoked ‚Äî nothing fires them automatically and you have to remember which fits the moment. This is the index. Tell me where you are with the material; I name the **one** skill to invoke and why, then hand off. I do not teach here ‚Äî I route.

## Route by where you are with the material

| You are‚Ä¶ | Invoke | Because |
| --- | --- | --- |
| **about to learn it ‚Äî unsure where to start** (your self-assessment is untested) | `baseline-prober` | probes what you actually know before any path, then keys one entry point to the result |
| **needing to be functional FAST** (hours, not a syllabus) | `fast-track` | the highest-leverage 20%, what to ignore, one keystone rep |
| **holding a concrete outcome + a deadline** | `plan-architect` | a day-by-day plan, one 45-minute task per day, traced back to the outcome |
| **drowning in jargon ‚Äî it won't click** (you do NOT understand it yet) | `confusion-translator` | finds the one keystone idea and translates it into plain life, gating on comprehension |
| **wanting to learn a concept by doing and failing, not by being told** | `learn-by-failing` | drops you in a realistic scenario, withholds the answer when you err |
| **fresh off learning it ‚Äî want to know if it actually stuck** | `explain-it-back` | you explain it to a 10-year-old; every gap gets stopped at, you repair it |
| **sure you've mastered it ‚Äî want that disproven** | `hidden-gaps-detector` | five deceptively simple questions + a blunt verdict on what your foundation is missing |
| **done learning ‚Äî want it to survive weeks, not hours** | `build-retention` | closed-book recall questions + a dated expanding-interval retrieval schedule |
| **prepping for a real interview, across sessions** | `interview-coach` | teaches the weak answer's technique, drills it from memory, keeps a standing workspace between sessions |

## Disambiguate the close pairs

- **`baseline-prober` vs `hidden-gaps-detector`** ‚Äî timing and intent. Before you learn, to find the right starting point: `baseline-prober`. After you claim mastery, to have it falsified: `hidden-gaps-detector`.
- **`confusion-translator` vs `explain-it-back`** ‚Äî direction. You cannot follow it yet: `confusion-translator` (the concept gets translated *to* you). You think you can already explain it: `explain-it-back` (you explain it and get stopped at every gap).
- **`hidden-gaps-detector` vs `learn-by-failing`** ‚Äî intent. You want claimed mastery falsified: `hidden-gaps-detector`. You want to build the skill through struggle: `learn-by-failing`.
- **`fast-track` vs `plan-architect`** ‚Äî time horizon. One sitting, usable now: `fast-track`. Multi-day, aimed at a checkable outcome by a date: `plan-architect`.

## How to hand off

State the matched skill by name and the one-line reason, then tell the user to invoke it. These skills are user-invoked ‚Äî this router cannot fire them. If two genuinely fit, name both and the deciding question above; do not pick silently. If none fit, say so ‚Äî do not force a learning skill onto a non-learning request.
