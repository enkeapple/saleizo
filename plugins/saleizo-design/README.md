# Design Kit

Skills for the parts of engineering that aren't a pipeline step: designing a module worth depending on, reviewing an architecture for where depth is missing, and reviewing changed code for quality. Prose de-slopping moved to its own plugin (`saleizo-prose`).

## Quickstart

- Designing or reshaping a module? Start with `codebase-design` for the shared deep-module vocabulary, then `improve-codebase-architecture` for a review pass.
- Reviewing a diff for reuse / simplification / quality cleanups? `auditing-code-quality`.
- Install it first — see [Installation](#installation).

## How it works

The skills are user-invoked: name the task ("design this module", "review this architecture") and Claude loads the matching skill. Each one carries a fixed vocabulary or catalog so the same pass runs the same way every time.

## Installation

### Claude Code

```text
/plugin marketplace add enkeapple/saleizo
/plugin install saleizo-design@saleizo
```

## What's Inside

- **codebase-design** — a shared vocabulary for designing deep modules (a lot of behaviour behind a small interface, at a clean seam).
- **improve-codebase-architecture** — an architecture-review pass that surfaces where interfaces are too shallow.
- **auditing-code-quality** — review changed code for reuse, simplification, and efficiency cleanups.

## Philosophy

Depth and clarity don't happen by default — they need a fixed pass the model's own taste keeps skipping. These skills encode that pass so it runs the same way every time, instead of relying on the agent to remember.
