# Prose Kit

Make text read like a person wrote it. Two passes remove the AI tells a model leaves behind — one for terse technical docs, one for narrative/article register — and a third turns a release diff into store-ready release notes.

## Quickstart

- Cleaning up a technical doc or weaker-model output? Reach for `tightening-prose`.
- Rewriting a draft into a publishable article / blog / post? `humanizing-prose`.
- Turning a release diff into Google Play / App Store notes? `drafting-release-notes`.
- Install it first — see [Installation](#installation).

## How it works

Each skill runs a fixed catalog the model's own taste reliably misses — phrase-level tells, sentence patterns, and a shared scoring rubric — so the cleanup is deterministic rather than dependent on what the agent happens to notice. The two prose passes have deliberately opposite carve-outs: `tightening-prose` keeps technical terseness, `humanizing-prose` cuts the narrative tells that terseness keeps.

## What's Inside

- **tightening-prose** — remove AI writing tells from an existing chunk of technical prose by running a fixed catalog.
- **humanizing-prose** — rewrite a draft into natural, publication-ready prose for a narrative / article register.
- **drafting-release-notes** — turn a release diff into user-facing store release notes (Google Play / App Store).

## Philosophy

A strong model's fresh output is already clean; these skills target the tells it cannot see in existing text — its own, a weaker model's, or a human's slop. A fixed catalog beats freehand editing because the same tells recur and freehand passes miss the same ones every time.
